# powerh — твій персональний тренер і медичний аналітик на власному ноутбуці

**powerh** — це невелика програма, яку ти запускаєш у себе на комп'ютері. Вона тримає всі твої тренування зі Strava, твої аналізи з PDF-файлів та твої цілі в одному місці й дає тобі особистого асистента на основі Claude (це найрозумніша AI-модель від Anthropic). Цей асистент бачить твою картину цілісно — як ти бігаєш, як спиш, який у тебе вітамін D — і може відповісти на питання типу "як минув мій тиждень", "що мені робити в четвер" або "чи помітні зміни в моїх аналізах за пів року".

> Це не сервіс і не підписка. Жодна твоя інформація нікуди не відправляється, крім запитів до Claude API — і за них платиш ти сам, копійки за розмову. Усі дані лежать у файлі на твоєму комп'ютері.

---

## Що це таке і чому це круто

Уяви, що в тебе є приватний тренер, який:

- **читав твої аналізи** — усі, які ти йому даси, у вигляді PDF-ів від лабораторії;
- **бачить кожне твоє тренування** — автоматично підтягує зі Strava;
- **пам'ятає попередні розмови** — і важливі факти про тебе (ти любиш ранкові пробіжки, у тебе хронічно низький феритин, ти готуєшся до півмарафону…);
- **доступний 24/7** — у твоєму браузері, без черг і записів;
- **щотижня сам пише тобі огляд** — короткий підсумок тижня з пропозицією на наступний.

Це не "ChatGPT із доступом до інтернету". Це асистент, у якого є **твої власні дані** — твій пульс, твоя HbA1c, твій тижневий обсяг бігу — і він може робити висновки на їх основі. Між "як зміцнити серце" та "ось як зміцнити **твоє** серце, бо твій тижневий обсяг останнім часом 17 км, а вітамін D 23" — велика різниця.

І найголовніше: усе це працює на твоєму ноутбуці. Ти не залежиш від чужого сервера. Не платиш підписку. Не довіряєш свої аналізи стороннім компаніям. Ти володар своїх даних — і свого асистента.

## Що потрібно для запуску

1. **Комп'ютер з macOS, Linux або Windows.** Будь-який сучасний.
2. **Node.js 20 або новіший** — це безкоштовна штука, яка дозволяє запускати програми, написані на JavaScript. Завантажується одним кліком з [nodejs.org](https://nodejs.org/).
3. **Anthropic API ключ** — щоб мати доступ до Claude. Створюється за 2 хвилини, потрібна банківська картка для поповнення балансу. Зазвичай $5 на рахунку вистачає на місяць активного користування.
4. **(За бажанням) Strava аккаунт** — якщо хочеш, щоб асистент бачив твої тренування.

Усе. Жодних серверів, доменів, ніяких "налаштувати DNS". Один раз клонуєш репозиторій, один раз вставляєш ключі — і користуєшся.

## Встановлення — крок за кроком

### Крок 1: Встанови Node.js

Зайди на [nodejs.org](https://nodejs.org/) і завантаж **LTS-версію** (зеленувата кнопка зліва). Запусти інсталятор, погоджуйся з усім за замовчуванням. Готово.

Щоб перевірити, що все встановилось, відкрий термінал (на Mac: Spotlight → "Terminal"; на Windows: меню Пуск → "PowerShell") і вбий:

```bash
node --version
```

Має показати щось типу `v20.x.x` або `v22.x.x`. Якщо так — все ОК.

### Крок 2: Завантаж powerh

У тому ж терміналі:

```bash
git clone <посилання_на_цей_репозиторій> powerh
cd powerh
npm install
```

Команда `npm install` встановить усі залежності — це займе 30–60 секунд. Можна йти заварити чай.

> Якщо в тебе немає `git` — встанови його з [git-scm.com](https://git-scm.com/) (для Windows та Linux) або через Xcode Command Line Tools (для Mac: `xcode-select --install`).

### Крок 3: Запусти powerh

Один рядок в одному терміналі:

```bash
npm start
```

Це запустить **обидві частини** powerh — backend (сервер з твоїми даними та API) і frontend (інтерфейс у браузері) — в одному вікні. Логи обох будуть з префіксами `[server]` і `[web]`, щоб ти бачив що відбувається.

Зачекай 5–10 секунд, поки обидві частини стартують. Коли побачиш у логах щось типу:

```
[server] powerh server ready
[web]    VITE ready in 800 ms
[web]    ➜  Local:   http://localhost:5173/
```

Відкривай у браузері: **http://localhost:5173**

Має з'явитися інтерфейс powerh з боковою панеллю — Чат, Огляд, Джерела, Медичні дані, Налаштування.

### Крок 4: Зупинити програму

У терміналі, де запущено `npm start`, натисни **`Ctrl+C`** один раз. Обидві частини зупиняться разом.

Щоб запустити знову — повтори Крок 3.

> 💡 **Якщо потрібно запустити частини окремо** (для розробки): `npm run dev:server` в одному терміналі та `npm run dev:web` в іншому. Або, якщо звик до `foreman`/`overmind` — у корені є `Procfile`.

### Що, якщо щось не запускається?

| Симптом | Спробуй |
|---|---|
| "Address already in use" / "port 5174 is busy" | Хтось уже слухає порт. На Mac/Linux: `lsof -ti:5174 \| xargs kill -9`. Або просто перезавантаж комп. |
| "command not found: npm" | Node не встановлений або не в PATH. Перевстанови з [nodejs.org](https://nodejs.org/). |
| "command not found: git" | Встанови з [git-scm.com](https://git-scm.com/). На Mac: `xcode-select --install`. |
| `npm install` падає з помилкою | Видали `node_modules/` і `package-lock.json`, запусти `npm install` знову. |
| Сервер запустився, але вкладка показує помилку | Спробуй жорстке оновлення вкладки: `Cmd+Shift+R` (Mac) / `Ctrl+Shift+R` (Windows/Linux). |
| Strava видає "Authorization Error" після натискання Connect | Зайди на [strava.com/settings/api](https://www.strava.com/settings/api), перевір що "Authorization Callback Domain" встановлено саме як `localhost` (без `http://`, без портів). |

## Налаштування — крок за кроком

### Anthropic API ключ (обов'язково)

1. Зайди на [console.anthropic.com](https://console.anthropic.com/) і зареєструйся (Google або email).
2. У лівому меню обери **Settings → API Keys** (Налаштування → API ключі).
3. Натисни **Create Key**, дай їй будь-яку назву ("powerh"), скопіюй ключ. **Увага: ключ показується тільки один раз** — скопіюй і збережи десь.
4. Тобі потрібно поповнити баланс — у меню **Billing** (Оплата). Додай $5–10 для початку. Цього вистачить на тижні-два активного користування.
5. У powerh відкрий **Налаштування**, встав ключ у поле "API ключ", натисни **Зберегти**.

Усе. Тепер можна йти в Чат і починати спілкуватися з Claude.

### Strava (необов'язково, але дуже рекомендую)

Без Strava асистент не бачитиме твоїх тренувань. Це налаштовується раз і займає 3 хвилини.

1. Зайди на [strava.com/settings/api](https://www.strava.com/settings/api).
2. **Create & Manage Your App.** Заповни:
   - **Application Name:** будь-що, наприклад `powerh`
   - **Category:** `Other`
   - **Club:** залиш порожнім
   - **Website:** `http://localhost:5174`
   - **Authorization Callback Domain:** `localhost` (тільки слово, без `http://`, без порту, без слешів)
3. Натисни **Create**. Може попросити завантажити іконку — постав будь-яку картинку.
4. Тепер ти побачиш свої **Client ID** (число) та **Client Secret** (рядок, треба натиснути "Show" щоб побачити).
5. У powerh відкрий **Налаштування**, встав обидва в розділі Strava, натисни **Зберегти**.
6. Йди в **Джерела**, натисни **Під'єднати Strava**. Strava запитає авторизацію, погоджуйся.
7. Тебе поверне до powerh. Натисни **Синхронізувати** — програма підтягне твої тренування за останні 90 днів.

Готово. Тепер усі майбутні твої пробіжки/велозаїзди/плавання автоматично з'являтимуться в powerh, як тільки натиснеш "Синхронізувати".

### Завантаження аналізів

У вкладці **Медичні дані** натисни на поле вибору файлу й завантаж PDF з результатами аналізів (наприклад, з Synevo, Сінево, ESC Lab чи будь-якої іншої лабораторії). Claude прочитає PDF, видобуде всі маркери (холестерин, гемоглобін, тестостерон тощо) і збереже їх у структурованому вигляді. Можна редагувати окремі значення, якщо Claude щось неправильно прочитав.

Можеш завантажувати скільки завгодно файлів — асистент бачитиме тренди ("LDL у тебе впав з 3,2 до 1,8 за останні 6 місяців").

## Як це працює, якщо ти не програміст

Ось спрощена картинка:

```
Ти              ↔   Браузер   ↔   powerh (на твоєму компі)   ↔   Claude API
                                       ↕
                                  ~/.powerh/powerh.sqlite
                              (один файл з усіма даними)
                                       ↕
                              ~/.powerh/uploads/
                              (твої PDF аналізів)
```

**Що це означає:**

- **Дані живуть у тебе.** Усе зберігається в одному файлі `powerh.sqlite` у твоїй домашній теці. Хочеш почати з чистого аркуша — видали цей файл, і все.
- **Strava → твій комп.** powerh раз на сесію бере твої нові тренування зі Strava (через офіційний API) і кладе їх у твій файл. Ніхто третій не задіяний.
- **PDF аналізи → Claude → твій комп.** Коли ти завантажуєш PDF, powerh відправляє його в Anthropic, Claude його читає, повертає структуру (маркер, значення, одиниці, межі норми) — і ця структура зберігається в тебе. Сам PDF теж у тебе.
- **Кожне повідомлення в чаті йде до Claude API.** Claude бачить твоє запитання + системну інструкцію + (якщо потрібно) витяги з твоїх даних, які він сам собі запитує через "інструменти". Відповідь стримається до тебе в реальному часі.
- **Розмови зберігаються.** Усі чати, які в тебе були з асистентом, лежать у тому самому файлі — можеш повернутися до них через бокову панель.
- **Асистент має пам'ять.** Він пише собі нотатки про важливі факти ("Олександр готується до півмарафону у вересні", "хронічно низький вітамін D"), і ці нотатки доступні йому в будь-якій майбутній розмові.

**Скільки інформації відправляється "назовні"?** Тільки те, що Claude потрібно прямо зараз, щоб відповісти. Він не "вивантажує" твої дані кудись. Anthropic не зберігає твої запити для тренування моделей (це їхня політика). Технічно — питай у Anthropic, але архітектурно ситуація проста: твій ноутбук → їхній API → відповідь.

## Хочеш не лише користуватися — хочеш **будувати**? Встанови Claude Code

Це найцікавіша частина. powerh — це **відкритий код**, і ти можеш його змінювати під себе. Хочеш додати інше джерело даних? Інші запитання? Іншу логіку щотижневого огляду? Іншу мову? Усе можна.

І тобі **не потрібно бути програмістом**. Тобі потрібен **Claude Code** — це консольний інструмент від Anthropic, який сам пише код на основі твоїх інструкцій. Я цим самим powerh-ом і будував — описував у чаті чого хочу, він писав і запускав.

### Встановити Claude Code

Один рядок у терміналі:

```bash
npm install -g @anthropic-ai/claude-code
```

Потім зайди в теку проекту:

```bash
cd powerh
claude
```

Введи свій Anthropic ключ, коли попросить, і все. У тебе тепер є асистент, який бачить **весь код** powerh і може його редагувати, запускати, тестувати — за твоїми вказівками.

### Що з цим можна зробити (тут і зараз)

Ось ідеї, які реалістично втілити, кажучи Claude Code українською:

- **"Додай поле «настрій» у щоденнику тренувань, щоб я міг записувати 1–5 після кожного забігу."**
  Claude Code змінить схему бази даних, додасть форму на сторінці, оновить асистента, щоб він враховував настрій. 10 хвилин.

- **"Зроби, щоб щоранку асистент сам присилав мені пропозицію тренування на сьогодні в Telegram."**
  Claude Code напише інтеграцію з Telegram-ботом, додасть планувальник. Півгодини.

- **"Я хочу бачити графік мого ваги/пульсу/вітаміну D за останні 6 місяців на головному екрані."**
  Графіки в Огляді. Стандартна задача. 15 хвилин.

- **"Підтягуй дані з Garmin Connect замість Strava."**
  Окремий адаптер. Не тривіально, але працює — Claude Code знає всі публічні API.

- **"Хочу, щоб асистент розмовляв зі мною голосом."**
  Інтеграція з Whisper для розпізнавання і ElevenLabs для синтезу. Реально за вечір.

- **"Зроби мобільну версію інтерфейсу."**
  Адаптивна верстка — Claude Code сам це робить.

- **"Додай можливість запитувати асистента про харчування — інтегруй з якимось API трекером калорій."**
  Інтеграція + новий інструмент для асистента. Кілька годин.

**Це не магія.** Це інструмент, який пише код за тебе, але ти маєш ним керувати — описувати, що хочеш, дивитися, що він зробив, тестувати. Це нова форма роботи — щось між замовником і інженером.

### Як з цим почати, якщо ніколи не писав код

1. Запусти `claude` у теці проекту.
2. Опиши українською, що хочеш змінити: *"Хочу, щоб на сторінці Огляд був графік мого тижневого обсягу бігу за останні 12 тижнів."*
3. Claude Code запропонує план, покаже файли, які змінюватиме, і запитає підтвердження.
4. Він зробить зміни, запустить програму, покаже, що вийшло.
5. Якщо щось не так — кажи, що саме не так. Ітеруй.

Це геть інший досвід, ніж "чат з AI". Це справжній помічник, який має руки в твоєму проекті.

## Часті проблеми

**"Запустив `npm run dev:server`, але нічого не відбувається."**
Перевір, що ти в теці `powerh` (`pwd` має показати шлях, що закінчується на `/powerh`). Якщо все ок — переконайся, що порт 5174 не зайнятий іншою програмою.

**"Підключив Strava, але активностей немає."**
Натисни **Синхронізувати** на сторінці Джерела. Перший раз тягне 90 днів — може зайняти хвилину.

**"Завантажив PDF, але маркерів немає."**
Перевір, чи це справді PDF з результатами аналізів (не сканована картинка, не направлення на аналіз, а саме результати). Якщо так — у терміналі сервера буде діагностичне повідомлення з причиною.

**"Asistant відповідає англійською."**
Системна інструкція написана так, щоб він відповідав українською, але якщо твоє повідомлення короткою фразою без контексту — може зіскочити. Просто попроси його продовжувати українською.

**"Хочу почати з чистого аркуша."**
Видали теку `~/.powerh/` (це твоя домашня тека → `.powerh`). Перезапусти сервер — він створить її заново.

## Архітектура (для технарів)

```
apps/
├── server/      Fastify + better-sqlite3 + @anthropic-ai/sdk
│   ├── agent/   tool-use loop, системна інструкція
│   ├── db/      схема, репозиторії, міграції
│   ├── sources/ strava (OAuth + sync), medical (PDF → структура)
│   ├── routes/  /api/{chat,setup,health,sources,medical,notes,plans,digest,conversations}
│   └── jobs/    node-cron щотижневий огляд
└── web/         Vite + React (Чат, Огляд, Джерела, Медичні, Налаштування)
```

Свідомо одна-користувацька установка. SQLite, без черг, без воркерів — щоб запустити можна було за 3 хвилини. Будь-яка інтеграція — окремий адаптер у `sources/`.
